Customer Support System

Resolve

Customer support operations platform with automated resolutions, multi-provider refunds, and a full review workflow — tightly integrated with the order lifecycle.

Full Stack BÆRSkin Tactical · DivBrands Production Microservice
◈ Overview

Resolve is a full-stack customer support operations platform that enables support teams to handle order modifications, cancellations, and refunds through a structured workflow. It integrates deeply with the existing order lifecycle, ensuring every operation is auditable, reversible, and reviewed before being committed.

The project demonstrates strong understanding of customer support flows, multi-provider payment integrations, and the importance of rollback safety in operations that touch live orders and financial records.

◈ Features
  • API for receiving operations requests from customer support departments
  • Internal UI to generate tickets, track processes, and review pending operations
  • Warehouse and stock management tied to item changes and cancellations
  • Item change and order cancellation operations with inventory reconciliation
  • Refund operations with Stripe, PayPal, Klarna, and Adyen integrations
  • Logging system to keep a full audit trail of all CS department operations
  • Review system requiring approval before committing changes to the system
  • Rollback logic to safely undo operations in case of downstream failures
◈ Architecture
System Context
Part of a broader microservices ecosystem. Resolve coordinates with the order, fulfillment, and payment services to execute support operations safely — emitting events consumed by downstream services.
Frontend
Internal UI for customer support teams to create tickets, review pending operations, and submit or reject changes before they're committed.
Backend API
Handles requests from the frontend and other services. Manages the ticket lifecycle, review gates, and integration calls to payment providers.
Async Workers
Inngest functions execute approved operations asynchronously — refunds, stock adjustments, cancellations — with retry logic and rollback handlers.
Database
PostgreSQL stores tickets, operation logs, review states, and refund records with transactional guarantees for audit integrity.
